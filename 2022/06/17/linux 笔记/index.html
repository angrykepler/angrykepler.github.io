1、网络连接的三种模式

一共有255 可用

192.168.0.1 网关

192.168.0.5 网络地址

1、桥接模式，虚拟系统可以和外部系统通讯，但是容易造成IP冲突

张三：192.168.0.20

虚拟系统Linux 192.168.0.~ 同一个网段，才能与外部通讯

2、NAT模式，网络地址转换模式，虚拟系统可以和外部系统通讯，不造成IP冲突

3、主机模式：独立的系统

2、配置nat模式流程

  1. 首先在vmwa中的虚拟网络编辑器中自定义一个nat模式，点击NAT设置，记录网关地址，以及DHCP设置，记录ip使用范围

  2. 进入虚拟机，在有线连接中配置IP4，将上述参数配置，dns:8.8.8.8,114.114.114.114

  3. 右键虚拟机标签设置，更改网络配置为，自定义的NAT配置

  4. ping 测试结果

### 安装阶段

1、安装`mysql`

`yum install mysql-community-server
 （如果检索密钥错误解决方法： rpm --import https://repo.mysql.com/RPM-GPG-KEY-mysql-2022）`

2、启动`MySQL`服务
`systemctl start mysqld`

3、查看`MySQL`的启动状态

`shell>   systemctl status mysqld`

4、设置开机启动`MySQL`

`shell> systemctl enable mysqld`
`shell> systemctl daemon-reload`

5、修改root本地登录密码
mysql安装完成之后，在/var/log/mysqld.log文件中给root生成了一个默认密码。通过下面的方式找到root默认密码，然后登录mysql进行修改： `shell> grep 'temporary password' /var/log/mysqld.log`

### Linux设置`MySQL`修改密码

1. 修改root本地登录密码

   `grep 'temporary password' /var/log/mysqld.log`

2. 登录并修改密码

   `mysql -uroot -p`

3. 修改密码（有要求）

   `ALTER USER 'root'@'localhost' IDENTIFIED BY 'newpassword'`

### Linux设置`MySQL`远程登录

1. 授权其他机器登陆

   mysql默认只允许root帐户在本地登录，如果要在其它机器上连接mysql，必须修改root允许远程连接，或者添加一个允许远程连接的帐户

   `GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY 'newpassword' WITH GRANT OPTION;`

2. 重新登录

   `mysql -uroot –p`

## 10、`Linux`防火墙开启

`开放3306端口(如下命令只只针对centos7以上)`
   `查看已经开放的端口：firewall-cmd --list-ports`
   ` 开启端口:firewall-cmd --zone=public --add-port=80/tcp --permanent`
`重启防火墙`
  `firewall-cmd --reload #修改端口后需要重启`
`firewall systemctl stop firewalld.service #停止`
`firewall systemctl disable firewalld.service #禁止firewall开机启动`
